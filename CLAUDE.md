# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a collection of TypeScript Design Pattern implementations, each as a standalone browser-runnable demo. Patterns are organized into three categories: Behavioral, Creational, and Structural. A Hub app (port 9001) aggregates all patterns into a single browsable UI.

## Commands

Each pattern lives in its own subdirectory with its own `package.json`. Commands must be run from within the pattern's directory (e.g., `cd Behavioral/ChainOfResponsibility`):

```bash
npm install       # Install dependencies
npm run build     # Compile TypeScript via webpack → dist/
npm start         # Build and launch webpack-dev-server at localhost:9000
```

Hub runs on port 9001:

```bash
cd Hub && npm install && npm start
```

There are no tests (the test script exits with an error by design).

## Architecture

Every pattern follows an identical structure:

```
PatternName/
  src/
    index.ts              # Entry point: wires up DOM events and pattern instances
    template.html         # HTML template (webpack reads this; never has a <script> tag)
    interfaces/           # TypeScript interfaces and abstract base classes
    models/               # Concrete implementations of the interfaces
    types/                # Shared type definitions (present in some patterns)
  index.html              # Built output — generated by webpack, do not edit directly
  style.css               # Pattern-specific styles; imports shared/base.css at the top
  webpack.config.js       # 2-line file: delegates to shared/webpack.base.js
  tsconfig.json           # 1-line file: extends shared/tsconfig.base.json
  package.json
```

The `index.ts` entry point always connects DOM elements to the pattern implementation. The `interfaces/` folder defines the contracts (abstract classes or TypeScript interfaces), and `models/` contains the concrete classes that fulfill them.

TypeScript is compiled by `ts-loader` through webpack. The `dist/` output is not committed. All patterns use `strict: true` with `strictPropertyInitialization: false`.

## Shared Modules (`shared/`)

All 21 patterns share three files under `shared/`. **Do not duplicate these into individual patterns.**

### `shared/tsconfig.base.json`

Base TypeScript config extended by every pattern. Contains all compiler options except path-sensitive fields (`rootDir`, `outDir`) which are omitted so they resolve correctly per-pattern.

Each pattern's `tsconfig.json` is just:
```json
{ "extends": "../../shared/tsconfig.base.json" }
```

### `shared/webpack.base.js`

Factory function used by every pattern's `webpack.config.js`:

```js
const createConfig = require("../../shared/webpack.base.js");
module.exports = createConfig(__dirname);
```

Accepts an optional second argument for overrides:
```js
module.exports = createConfig(__dirname, { template: "./src/template.html", port: 9000 });
```

The factory:
- Resolves `html-webpack-plugin` and loaders from the **pattern's own** `node_modules` (via `module.createRequire`) — do not add these packages to `shared/`
- Outputs to `patternDir/dist/index.js` and `patternDir/index.html`
- Serves `shared/` at `/shared` so `@import "/shared/base.css"` resolves in dev

### `shared/base.css`

Common reset and layout styles imported by every pattern's `style.css`. Uses CSS custom properties for accent theming:

```css
@import "/shared/base.css";

:root {
  --badge-bg:   rgba(99,102,241,0.15);  /* set per-pattern */
  --badge-cl:   #a5b4fc;
  --badge-bd:   rgba(99,102,241,0.3);
  --flow-bg:    rgba(99,102,241,0.04);
  --node-bg:    rgba(99,102,241,0.08);
  --node-bd:    rgba(99,102,241,0.2);
  --node-cl:    #818cf8;
  --node-hi-bg: rgba(99,102,241,0.2);
  --node-hi-bd: rgba(99,102,241,0.5);
  --node-hi-cl: #a5b4fc;
}
```

**Accent themes by category:**
- Behavioral + Adapter: indigo (`#6366f1` family)
- Creational: emerald (`#10b981` family)
- Structural (except Adapter): amber (`#f59e0b` family)

The `@import "/shared/base.css"` URL resolves because:
- Standalone dev server (port 9000): webpack.base.js serves `shared/` at `/shared`
- Hub dev server (port 9001): serves the repo root at `/`, so `/shared/base.css` maps directly

## HTML Templates

**All 21 patterns use `src/template.html`** as the webpack template (never `index.html`). Webpack reads the template, injects the `<script>` tag, and writes the result to `index.html`. Never add `<script>` tags to `src/template.html` — they will be duplicated on every build.

The `index.html` in the pattern root is built output. To edit the page structure, edit `src/template.html`.

## Critical: HtmlWebpackPlugin and Duplicate Script Tags

`webpack.config.js` uses `HtmlWebpackPlugin` with `filename: "../index.html"`. If the same file is used as both template (`template:`) and output (`filename:`), every build re-reads the injected script tag and adds another one.

**Fix:** always use a separate `src/template.html` as the template. The `webpack.base.js` factory defaults to `"./src/template.html"` for this reason.

## Window Globals: Connecting TypeScript to HTML

Webpack bundles each pattern as an isolated module. Functions defined in `index.ts` are **not** reachable from HTML `onclick`/`onsubmit` attributes by default.

To expose a function to the HTML:
```ts
(window as any).myFunction = myFunction;
```

Every pattern that uses HTML event attributes follows this convention. When adding new functions called from HTML, always add the corresponding `window` assignment at the bottom of `index.ts`.

This also applies cross-module: if a model class needs to call a function defined in `index.ts`, it must call it via `(window as any).functionName()` — direct imports would create circular dependencies.

## CSS Display Mode Conflicts

When showing/hiding elements with `element.style.display`, match the value to the CSS rule for that element. Setting `"block"` on a flex container breaks its layout. Check `style.css` for the element's display rule before writing:

```ts
element.style.display = "flex";  // not "block" if CSS uses display: flex
```

## Hub

The Hub (`Hub/`) runs on port 9001 and serves all patterns in an iframe. Key points:

- `Hub/src/patterns.ts` is the registry. Set `hasDemo: true` for any pattern that has a built `index.html`.
- Hub's iframe uses `sandbox="allow-scripts allow-same-origin allow-forms allow-popups"`. Note: `allow-modals` is intentionally absent — patterns must not use `alert()` or `confirm()`. Use DOM-based result rendering instead.
- Hub's webpack config is **not** derived from `shared/webpack.base.js` — it has a unique devServer config that serves the entire repo root so iframe URLs like `/Behavioral/Strategy/index.html` resolve correctly.
- The Hub serves `shared/` at the root level automatically (since it serves the repo root), so `@import "/shared/base.css"` in pattern CSS files works without any Hub-specific config.

## Adding a New Pattern

1. Create a new directory under the appropriate category (`Behavioral/`, `Creational/`, or `Structural/`).
2. Create `tsconfig.json` with just `{ "extends": "../../shared/tsconfig.base.json" }`.
3. Create `webpack.config.js` with just the two-line factory call.
4. Copy `package.json` from an existing pattern and change the `name` field.
5. Create `src/template.html` (no `<script>` tag — webpack injects it).
6. Create `style.css` starting with `@import "/shared/base.css"` followed by the `:root` accent vars and pattern-specific styles.
7. Follow the `src/interfaces/` + `src/models/` + `src/index.ts` structure.
8. Expose any functions called from HTML via `(window as any).funcName = funcName` in `index.ts`.
9. Run `npm install && npm run build` to verify.
10. Set `hasDemo: true` in `Hub/src/patterns.ts` and rebuild the Hub.
